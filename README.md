# 🚀 Video Anomaly Detection and Alert System

This final year project (PFE) implements an intelligent system for video surveillance. The source code handles **anomaly detection** (such as falls, fights, or suspicious behaviors) using pose estimation with YOLOv8 models. Once an anomaly is detected by the vision model, a dedicated agent takes over to manage the **alert** system in real-time.

## 🖥️ User Interface

The project includes an interactive web dashboard built with Streamlit, allowing users to upload videos, run the detection models, and visualize the results in real-time.

![Streamlit Interface](assets/interface_streamlit.png)

## 📁 Project Architecture

The project is structured as follows:

* `app.py`: Interactive user interface (developed with Streamlit) to launch and visualize detections.
* `data_collection.py`: Script for collecting and preparing video data.
* `data_visualisations.py`: Tools for visualizing metrics and results.
* `model_training.py`: Script dedicated to training and fine-tuning the detection models.
* `yolov8n-pose.pt` / `yolov8s-pose.pt`: Pre-trained weights for YOLOv8 models (Nano and Small) for pose estimation.
* `requirements.txt`: List of dependencies and Python libraries required for the project.
* `workflows/n8n_alert_workflow.json`: Exported n8n workflow file for the automated alerting system.
* `dataset_pose/`, `models/`, `testing_videos/`, `local_saved_clips/`: Folders containing training data, saved models, test videos, and extracted anomaly clips.
* `assets/`: Folder containing project screenshots.

## 🔔 Alert System Integration (n8n)

When an anomaly is detected, the system sends an HTTP POST request to an **n8n webhook** to trigger an automated alert workflow. This workflow analyzes the video clip using an LLM (Google Gemini) and sends alerts via Email and Telegram.

### Workflow Overview:
![n8n Workflow](assets/n8n_workflow.png)

### Setting up the automated alerts:

1. **Import the Workflow:**
   * Open your n8n instance.
   * Go to workflows, click on **Import from File**, and select the `workflows/n8n_alert_workflow.json` file provided in this repository.
2. **Configure Credentials:**
   * Update the nodes with your own credentials (Google Gemini API key, Telegram Bot Token, and SMTP/Email credentials).
3. **Link the Application:**
   * Copy the **Test/Production Webhook URL** generated by the n8n Webhook node.
   * Paste this URL into your `app.py` or `.env` file where the detection script sends the POST request.

## 🛠️ Installation and Usage

1. **Clone the repository:**
   ```bash
   git clone <YOUR_REPO_LINK>
   cd PFE_Anomaly_Detection
   ```

2. **Install dependencies:**
   It is highly recommended to use a virtual environment.
   ```bash
   pip install -r requirements.txt
   ```

3. **Run the application:**
   ```bash
   streamlit run app.py
   ```
